📋 Suggested Workflow

  1. Module 2A (40 min): Open Bug Verification checklist → Select your team → Reproduce each bug step-by-step → Document your findings
  2. Break (10 min): Discuss findings with your team → Compare results
  3. Module 2B (50 min): Open Feature Test Planning → Select your team → Review suggested scenarios → Ask Claude Code for help → Create your test cases
  4. Export: Save your work using the export buttons → Share with team → Use in Module 3 & 4

💡 Tips for Success

  1. Use Claude Code: Don't work alone! Ask Claude to analyze requirements, suggest test cases, and identify edge cases you might miss.
  2. Think Like a User: Focus on GUI workflows and user experiences, not technical implementation.
  3. Document Everything: Use the notes sections to capture observations, variations, and additional findings.
  4. Save Frequently: Click "Save Progress" regularly. Your work is stored in browser local storage.
  5. Export Your Work: Use export buttons to save text files of your verification reports and test plans.
