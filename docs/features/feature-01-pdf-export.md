# Feature #1: Export Inspection Report as PDF

**Assigned To**: Team 1
**Priority**: High
**Component**: Inspection Details
**Type**: Reporting/Export Feature

## Business Value

Restaurant owners and managers need to share inspection reports with health authorities, print them for records, or send them via email to stakeholders. Currently, there's no easy way to export a professional-looking report.

This feature enables compliance documentation and professional reporting.

## User Story

**As a** restaurant owner or manager
**I want to** export completed inspection reports as PDF files
**So that** I can share them with health authorities, print them for records, or send them via email to stakeholders

## Acceptance Criteria

### 1. Export Button

- Add "Export as PDF" button on completed inspection detail pages
- Button is visible ONLY to managers and restaurant owners
- Button is disabled (greyed out) for inspections with status "in_progress"
- Button position: Top-right of inspection details section
- Button styling: Primary action button with PDF icon
- Hover state shows tooltip: "Download inspection report as PDF"

### 2. PDF Content

The generated PDF must include:

**Header Section**:
- Company/System logo (if available)
- Report title: "Inspection Report"
- Generation timestamp: "Generated on: [date] at [time]"

**Restaurant Information**:
- Restaurant name (prominent, large font)
- Full address (street, city, postal code)
- Phone number and email
- Restaurant status (active/inactive)

**Inspection Overview**:
- Inspection date
- Inspector name and contact
- Inspection type (if applicable)
- Overall score (prominently displayed with color-coding)
  - Green: 8.0 - 10.0
  - Yellow: 6.0 - 7.9
  - Red: < 6.0

**Detailed Breakdown**:
- Scores grouped by category (Kitchen, Bar, Main Hall)
- For each category:
  - Category name as section header
  - Table of items with scores
  - Average score for category
- Each item shows:
  - Item name
  - Description
  - Score (1-10)
  - Inspector notes (if any)

**Footer Section**:
- Page numbers (e.g., "Page 1 of 2")
- Disclaimer: "This is an automated report generated by the Restaurant Inspection System"
- Optional: Digital signature or verification code

### 3. File Naming Convention

- Format: `Inspection_[RestaurantName]_[Date].pdf`
- Date format: YYYY-MM-DD
- Replace spaces in restaurant name with underscores
- Examples:
  - `Inspection_Downtown_Pizza_2026-04-15.pdf`
  - `Inspection_Sunrise_Cafe_2026-04-20.pdf`

### 4. User Experience

- **Loading State**: Show loading spinner while PDF generates
- **Progress**: Display "Generating PDF..." message
- **Success**: Trigger browser download automatically
- **Success Notification**: Show toast/message "PDF downloaded successfully"
- **Error Handling**: If generation fails:
  - Show error message: "Failed to generate PDF. Please try again."
  - Provide "Retry" button
  - Log error for debugging

### 5. Technical Requirements

- PDF generation should complete within 5 seconds
- Generated PDF should be properly formatted for printing (A4 or Letter size)
- PDF should be accessible (searchable text, not image)
- File size should be reasonable (< 2MB for typical inspection)

## Out of Scope (Not Required)

- Email directly from the app (user can email the downloaded PDF manually)
- Custom branding/logo selection (use default logo or none)
- PDF password protection
- Multiple export formats (Excel, Word, etc.)

## Test Scenarios

### Functional Tests

1. **Happy Path**: Manager downloads PDF for completed inspection
2. **Owner Access**: Restaurant owner can download their own inspection PDF
3. **Inspector Access**: Inspectors should NOT see export button (or it's disabled)
4. **In-Progress Inspection**: Button disabled when inspection not completed
5. **Multiple Inspections**: Can download multiple PDFs without issues

### Content Verification

6. **All Required Data**: PDF contains all required sections
7. **Correct Scores**: Scores in PDF match dashboard scores
8. **Category Grouping**: Items properly grouped by category
9. **Notes Included**: Inspector notes appear in PDF
10. **No Missing Data**: No undefined/null values displayed

### Edge Cases

11. **Long Restaurant Names**: Name doesn't overflow in PDF layout
12. **Many Items**: Inspection with 50+ checklist items fits properly
13. **No Notes**: PDF handles inspections with no inspector notes
14. **Special Characters**: Restaurant name with (&, ©, ™) works
15. **Large Scores**: Handles all score ranges (1-10) correctly

### User Experience

16. **Loading Indicator**: Shows while generating (test with slow connection)
17. **Error Recovery**: Retry works after failure
18. **File Download**: PDF actually downloads to user's device
19. **File Opening**: Downloaded PDF opens correctly in PDF reader
20. **Print Quality**: PDF looks professional when printed

### Security & Authorization

21. **Owner Restriction**: Owner can only export their own restaurant's inspections
22. **Manager Access**: Manager can export any inspection
23. **Unauthenticated**: Logged-out users cannot access export
24. **Role Verification**: Inspector role properly restricted

## Non-Functional Requirements

- **Performance**: PDF generation < 5 seconds
- **Accessibility**: Button has proper ARIA labels
